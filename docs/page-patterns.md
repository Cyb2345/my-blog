# 后台页面模式

## 列表页

标准结构：

```tsx
<AdminPage title="页面标题" description="页面说明">
  <AdminSearchForm />
  <Card>
    <AdminTableToolbar />
    <AdminDataTable />
  </Card>
</AdminPage>
```

必须包含：

- 查询
- 重置
- 新增
- 批量删除
- 分页
- loading
- 空状态
- 行操作
- 状态标签
- 长文本省略和 tooltip/title

适用：文章管理、分类管理、标签管理、友链管理、文件列表、参数管理、字典管理、用户管理、角色管理、访问日志、操作日志。

## 配置页

配置页以分组卡片呈现，字段按业务域分组。保存按钮只出现一次，放在表单底部或固定操作区。配置项必须真实联动功能，不做纯展示参数。

适用：站点配置、首页配置、登录页配置、导航配置、文件配置、参数管理。

## 表单页

表单页必须使用统一字段组件：

- label 宽度和位置统一。
- 输入框高度统一。
- 必填字段有明确标识。
- 保存失败不关闭弹窗。
- 保存成功后关闭并刷新数据。

## 详情页

详情页以 `Card` + 描述列表呈现，禁止把详情伪装成不可编辑表格。长文本使用换行或可复制区域。

## 日志页

日志页默认只读，支持筛选、分页、详情查看和必要的删除确认。日志表格不使用复杂行级动画。

## 文件管理页

文件管理必须区分本地、S3、Cloudflare R2。

本地存储只显示：

- 配置名
- 存储路径
- 访问路径
- 自定义域名
- 基础路径
- 状态

R2 / S3 才显示：

- Access Key
- Secret Key
- Bucket
- Endpoint
- Region
- 公开访问域名
- 路径前缀

文件列表必须支持文件预览、复制 URL、存储器标签、批量删除和分页。

## 系统参数页

参数管理必须真实联动系统功能。系统内置参数可以编辑值，但不能删除。示例：

- 登录验证码类型影响登录页。
- 是否开启 MFA 影响登录流程。
- 最大上传图片大小影响上传接口。
- 默认主题模式影响首次访问主题。
- 是否开启留言影响前台留言入口。

## 迁移优先级

第一批：

1. 文章管理
2. 友链管理
3. 文件列表
4. 文件配置

第二批：

1. 参数管理
2. 字典管理
3. 导航配置
4. 站点配置

第三批：

1. 分类管理
2. 标签管理
3. 用户管理
4. 角色管理
5. 日志管理
